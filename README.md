###版本更新日志 (V1.0.2)
####更新日期：2026-03-07
####更新类型：功能新增 + 样式优化 + 代码重构
####关联模块：前端（Vue）、状态管理（Pinia）、接口集成
---
###一、前端核心更新
####1. 功能模块新增
- 右侧边栏基础架构：
实现与左侧栏规格统一的右侧边栏，支持响应式布局（移动端 md 尺寸下自动隐藏）
新增折叠 / 展开功能，支持 384px (w-96) ↔ 120px 宽度切换，配套平滑过渡动画
- 当前播放详情展示：
顶部展示大幅方形歌曲封面图，分行显示大字体歌曲名称与艺人名称
集成喜欢 / 取消喜欢功能按钮（位于歌曲信息右侧），完成鉴权逻辑与状态切换实现
- 动态 “关于艺人” 板块：
监听当前播放歌曲状态，自动调用 getAllArtists → getArtistDetail 接口获取艺人详情
展示艺人圆形头像、名称及简介，对简介文本做 HTML 标签剥离和多行截断处理
实现艺人信息点击跳转至 /artist/:id 艺人详情页的路由导航
增加数据缺失时的兜底展示（默认头像、“暂无介绍” 文本）
- “队列中的下一首” 预览：
实时计算并展示播放列表下一首歌曲信息（封面、歌名、歌手）
实现点击卡片直接切歌的 “点击即播” 功能
新增封面悬停时的播放图标遮罩交互效果
####2. 样式与界面优化
- 深色模式适配：全面兼容 dark 模式，背景色适配为 #121212/#1e1e1e，文字颜色自动适配
- 滚动条优化：内容区域使用自定义细滚动条（custom-scrollbar），优化界面整洁度
- 布局排版：参考 Spotify 右侧栏设计风格，采用卡片式排版，优化视觉层次与布局结构
####3. 代码优化
- 逻辑解耦：通过 computed 和 watch 响应式监听 AudioStore 和 UserStore 状态变化，提升代码可维护性
- 错误处理：在艺人信息获取、收藏操作等核心逻辑中增加 try-catch 异常捕获及 ElMessage 提示，优化用户体验
