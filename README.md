更新日志：右侧边栏功能实现
新增功能 
右侧边栏基础架构:
实现了与左侧栏规格一致的右侧边栏，支持响应式布局（在移动端 md 尺寸下隐藏）。
增加了折叠/展开功能，支持从 384px (w-96) 切换至 120px，包含平滑过渡动画。
当前播放详情展示:
顶部展示大幅方形封面图。
歌曲名称（大字体）与艺人名称分行显示。
集成了喜欢/取消喜欢功能按钮，位于歌曲信息右侧，并实现了鉴权与状态切换逻辑。
动态“关于艺人”板块 (About Artist):
实现了监听当前播放歌曲，自动调用 API (getAllArtists -> getArtistDetail) 获取艺人详情。
展示艺人圆形头像、艺人名称以及艺人简介。
对简介文本进行了 HTML 标签剥离 (replace) 和多行截断处理。
添加了点击艺人信息跳转至艺人详情页 (/artist/:id) 的路由导航。
增加了数据缺失时的兜底显示（默认头像、暂无介绍文本）。
“队列中的下一首”预览:
实时计算并展示播放列表中的下一首歌曲信息（封面、歌名、歌手）。
实现了点击即播功能（点击卡片直接切歌）。
添加了悬停主要封面时的播放图标遮罩效果。
样式与界面 
深色模式适配: 全面支持 dark 模式，背景色适配为 #121212/#1e1e1e，文字颜色自动调整。
滚动条优化: 内容区域使用了自定义样式的细滚动条 (custom-scrollbar)，保持界面整洁。
布局排版: 此布局结构和视觉层次参考了 Spotify 的右侧栏设计，采用卡片式排版。
代码优化 
逻辑解耦: 使用 computed 和 watch 自动响应 AudioStore 和 UserStore 的状态变化。
错误处理: 在获取艺人信息和收藏操作中增加了 try-catch 及 ElMessage 提示，提升用户体验。
